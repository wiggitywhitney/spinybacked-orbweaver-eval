### Summary
The developer updated the telemetry agent specification document (v3.1) with a detailed redesign of cost visibility and confirmation mechanisms. The key changes focus on transforming the pre-run cost estimation from a passive PR summary feature to an active, configurable pre-run confirmation step.

The specification now introduces a more transparent approach to token usage, explicitly renaming "estimates" to "ceilings" to reflect the conservative nature of the pre-run calculation. A new `onCostCeilingReady` callback allows interface layers to surface cost information and request user confirmation before running the agent. This change adds a configurable `confirmEstimate` option that defaults to true, with CLI support for a `--yes` flag to bypass confirmation.

The developer carefully designed the cost visibility to work across different interfaces:
- CLI: Prompts user for confirmation when `confirmEstimate` is true
- GitHub Action: Always uses `--yes` (non-interactive)
- MCP server: Introduces a two-tool flow with `get-cost-ceiling` and `instrument` tools to handle confirmation at the tool boundary

The specification also adds an exit code 3 for user-aborted runs and clarifies how the cost ceiling is calculated. The PR summary will continue to show both the pre-run ceiling and actual token usage, helping users calibrate expectations for future runs.

These changes address the core limitation of the previous approach - a pre-run "estimate" that was only visible after the fact - by making cost visibility an active part of the instrumentation workflow.
