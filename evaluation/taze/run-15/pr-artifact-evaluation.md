**Advisory note on CDQ-006 exemption claim**: The PR's advisory message includes "Skip this finding for root spans at entry points — the guard adds clutter for negligible gain there." This is an advisory override suggested by the PR description, but it is NOT a rubric exemption. The rubric has no entry-point exception for CDQ-006. The rubric failures for checkGlobal.ts, interactive.ts, bunWorkspaces.ts, and pnpmWorkspaces.ts stand regardless of the advisory text.

### CDQ-007 Advisories

CDQ-007 (PII attribute names, raw filesystem paths) fired widely across 9 files. CDQ-007 is an advisory rule — it does not affect rubric scores unless a specific canonical violation is confirmed. The most likely trigger is `taze.write.file_path` attribute (set in packageJson.ts, packageYaml.ts, packages.ts, bunWorkspaces.ts, pnpmWorkspaces.ts, api/check.ts), which stores full file paths. The advisory suggests using basename instead of the full path. This is noted as a per-file advisory finding but is not a canonical CDQ-007 rubric failure in this evaluation (the rubric does not mandate basename-only paths; this is advisory guidance).

### SCH-001 Advisories

SCH-001 (Span Names Match Registry) fired for: packageJson.ts, packageYaml.ts, packages.ts, pnpmWorkspaces.ts, api/check.ts, packument.ts.

This is a **live-check tool limitation**: the validator evaluates span names against the schema state at the time the live-check was run, which may pre-date the schema additions committed in this run. All 27 span IDs are correctly listed in the Schema Changes section of the PR (they were added to agent-extensions.yaml by the agent), and per-file evaluation confirms SCH-001 PASS for all 11 files. The SCH-001 advisory is a tooling artifact, not a rubric violation.

### "Outlier" Flag — packages.ts

The PR flags packages.ts as "outlier, review recommended" (5 spans, average 2 across the file set). This is accurate — packages.ts has the highest span count of any file.
